# Migrating to v0.13: selection and folder-structure flags

v0.13 reshaped how you tell kei *what* to download and *where* to put it. The old v0.13 compatibility aliases were removed in v0.20.

On v0.20 and later, put persistent settings in TOML instead of sync flags.
See [v0.20 migration guide](v0.20-migration.md) and
[example.config.toml](../example.config.toml).

## What you actually need to do

- If you use `--exclude-album`, switch to `--album '!NAME'`. The old flag was removed in v0.20. The comma-separated form (`--exclude-album A,B` or `KEI_EXCLUDE_ALBUM=A,B`) does not carry over: `--album` does not split on commas, so use one flag per name (`--album '!A' --album '!B'`) or the TOML array form (`[filters].albums = ["!A", "!B"]`).
- If you have `{album}` in `--folder-structure`, move it to `--folder-structure-albums`. The auto-migration warning was removed in v0.20.
- If you ran `kei sync` with no `-a` flag and no `{album}` in your `--folder-structure`, v0.12 ran a single library-wide enumeration. v0.13 runs one enumeration per album plus one unfiled pass. With the default `--folder-structure %Y/%m/%d` (no `{album}`), on-disk paths stay identical to v0.12. The startup log line `Sync plan for library` reports the concrete pass count.
- v0.12 configs that already used `-a all` (or had `{album}` in the template, which auto-promoted to `-a all`) see no change in API surface; the new no-flag default matches what they were already running.

Everything else is additive in the v0.13 shape. On v0.20 and later, use the new names directly.

## Quick before/after

```sh
# Before (v0.12) - two named album passes, exclusion, album-prefixed dates
kei sync -a Vacation -a Family --exclude-album Drafts --folder-structure '{album}/%Y/%m/%d'

# After (v0.13) - same on-disk layout, plus an unfiled pass alongside
kei sync --album Vacation --album Family --album '!Drafts' \
         --folder-structure-albums '{album}/%Y/%m/%d'
```

The v0.13 form runs an additional unfiled pass by default (every photo not in any user album, written to `--folder-structure`'s default `%Y/%m/%d`). To match v0.12 exactly - just the named passes, nothing else - add `--unfiled false`:

```sh
kei sync --album Vacation --album Family --album '!Drafts' \
         --folder-structure-albums '{album}/%Y/%m/%d' \
         --unfiled false
```

kei warns once on startup whenever `--unfiled` is implicit and the album selector isn't `none`, pointing at this exact toggle.

```sh
# Separate template for unfiled photos (closes #215)
kei sync --folder-structure-albums '{album}' --folder-structure '%Y/%m'
# Albumed photos -> Vacation/IMG.JPG
# Unfiled photos -> 2024/06/IMG.JPG
```

## Selection flags

| Flag | Values | Default | Notes |
|---|---|---|---|
| `--album NAME` | literal name, `all`, `none`, `!literal`, `=literal` | `all` | Repeatable. `all` is every user album. Use `=all`, `=none`, or `=!Drafts` for literal names that look like sentinels. |
| `--smart-folder NAME` | literal name, `all`, `all-with-sensitive`, `none`, `!literal`, `=literal` | `none` | Repeatable. `all` excludes Hidden + Recently Deleted; `all-with-sensitive` includes them. Use `=` for literal sentinel-shaped names. |
| `--library NAME` | raw zone name (`PrimarySync`, `SharedSync-...`), truncated 8-char form, `all`, `none`, `primary`, `shared`, `!literal`, `=literal` | `primary` | Repeatable. Use `=` for literal zone names that look like sentinels. |
| `--unfiled` | `true`, `false` | `true` | Boolean. Bare `--unfiled` sets true; `--unfiled false` disables. |

Defaults apply per category. Setting `--smart-folder Favorites` doesn't change the `--album` default; you still get every album plus Favorites.

`!` prefix means exclude. Bare `--album '!Family'` means "every album except Family" (the `!` operates on the category default, which is `all`).

## Layout flags

| Flag | Default | Tokens accepted |
|---|---|---|
| `--folder-structure TEMPLATE` | `%Y/%m/%d` | strftime, `{library}` |
| `--folder-structure-albums TEMPLATE` | `{album}` | strftime, `{library}`, `{album}` |
| `--folder-structure-smart-folders TEMPLATE` | `{smart-folder}` | strftime, `{library}`, `{smart-folder}` |

`--folder-structure` is the unfiled-pass template. The two category templates default to flat per-folder layouts (`{album}` / `{smart-folder}`) because albums are already organised collections. Add date hierarchy explicitly: `--folder-structure-albums '{album}/%Y/%m/%d'`.

## Tokens

| Token | Allowed in | Placement | Renders as |
|---|---|---|---|
| `{library}` | every template | first segment, single occurrence | `PrimarySync` or `SharedSync-A1B2C3D4` (truncated 8-char UUID prefix) |
| `{album}` | `--folder-structure-albums` only | first segment, or second after `{library}` | album name |
| `{smart-folder}` | `--folder-structure-smart-folders` only | first segment, or second after `{library}` | smart-folder name |
| strftime (`%Y`, `%m`, `%B`, `%d`, ...) | every template | anywhere | as today |

Tokens used in the wrong template bail at startup with a clear error.

## Multi-library disambiguation

`--library` now takes multiple values. With more than one library in scope, put `{library}` in your templates to keep paths separate:

```sh
kei sync --library all --folder-structure-albums '{library}/{album}'
# Photos -> PrimarySync/Vacation/IMG.JPG
#       and SharedSync-A1B2C3D4/Vacation/IMG.JPG
```

Without `{library}` in the template, kei warns at startup that paths will share a namespace and the state DB will silently dedup cross-library matches. The warning is informational; the run continues.

## TOML mapping

`[filters]` extends with per-category arrays. `[download]` extends with per-category templates.

```toml
[filters]
albums = ["all", "!Family"]      # CLI: --album all --album '!Family'
smart_folders = []               # CLI: (no --smart-folder flags)
libraries = ["primary"]          # CLI: --library primary
unfiled = true                   # CLI: --unfiled

[download]
folder_structure = "%Y/%m/%d"
folder_structure_albums = "{album}"           # flat per-album default
folder_structure_smart_folders = "{smart-folder}"
```

These old keys worked through v0.19 with a warning and were removed in v0.20:

| Old key | New key |
|---|---|
| `[filters].album` | `[filters].albums` |
| `[filters].exclude_albums` | merge into `[filters].albums` as `"!name"` entries |
| `[filters].library` (single string) | `[filters].libraries` (array) |

## Removed / renamed in v0.20

| Old | Replacement | Status |
|---|---|---|
| `--exclude-album NAME` | `--album '!NAME'` | removed in v0.20 |
| `{album}` in `--folder-structure` | `--folder-structure-albums "{album}/..."` | removed in v0.20 |
| Auto-`-a all` from `{album}` token in template | covered by new default `--album all` | removed in v0.20 |
| `[filters].album` (TOML) | `[filters].albums` | removed in v0.20 |
| `[filters].exclude_albums` (TOML) | `[filters].albums` with `!` entries | removed in v0.20 |
| `[filters].library` (TOML, single string) | `[filters].libraries` (array) | removed in v0.20 |

## Behavior changes worth knowing

- **`kei sync` with no flags** runs N+1 enumerations (one per album, one unfiled) instead of v0.12's single library-wide pass. With the default `--folder-structure %Y/%m/%d` (no `{album}`), on-disk paths are unchanged. See "What you actually need to do" above for the full breakdown.
- **`--unfiled` defaults to `true` regardless of `--album`.** `kei sync --album Vacation` runs the Vacation pass *and* the unfiled pass. To restrict to just Vacation, add `--unfiled false`. kei prints a one-line warning at startup whenever `--unfiled` is implicit and the album selector isn't `none`, pointing at the toggle.
- **`--smart-folder all` skips Hidden + Recently Deleted** (mirroring iOS Photos). Use `--smart-folder all-with-sensitive` for literal everything.
- **A photo in N albums writes N on-disk copies** (unchanged from v0.12) when the active `--folder-structure-albums` template renders distinct paths per `{album}`. Photos in any album are excluded from the unfiled pass.
- **Literal names that look like selector tokens** need a leading `=`. Use `--album =all`, `--album =none`, or `--album '=!Drafts'` for albums with those exact names. The same escape works for smart folders and libraries.

## State DB schema v9

The `assets`, `asset_albums`, and `asset_people` tables all gain a `library` column in their primary key so per-library rows -- including album and person memberships -- stay distinct across PrimarySync and shared zones. Migration runs automatically on first sync after upgrade and backfills existing rows to `library = "PrimarySync"`. Single-library users see no behavior change. The backfill is exact for any database upgrading from a released kei since multi-library writes were never wired before this release.

## Validation

| You did this | kei does this |
|---|---|
| `--album Vacatiom` (typo, no such album) | Bails at startup: `Album 'Vacatiom' not found. Available albums: [...]`. |
| `--album '!Vacatiom'` (typo in exclusion) | Bails at startup: `Excluded album 'Vacatiom' not found. Available albums: [...]`. Stops a typo from silently matching nothing. |
| `--album Favorites` (passing a smart folder to `--album`) | Bails: `'Favorites' is a smart folder; pass --smart-folder Favorites instead of --album`. |
| `--album Foo --album '!Foo'` | Bails at parse time. |
| `--library none` | Bails: `'--library none' is not allowed: kei needs at least one source library to sync`. |
| `--album none --smart-folder none --unfiled false` (everything turned off, valid library) | Config builds; the sync exits without doing any download work. |
| `{album}` in `--folder-structure-smart-folders` (or `{smart-folder}` in `--folder-structure-albums`) | Bails at startup. |
